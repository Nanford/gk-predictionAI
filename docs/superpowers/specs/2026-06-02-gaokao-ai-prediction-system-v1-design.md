# 高考志愿 AI 预测系统 1.0 产品设计

## 1. 产品定位

高考志愿 AI 预测系统是一款面向高考考生与家长的志愿填报辅助参考工具。系统基于省份报考规则、考生分数与位次、近三年录取数据、招生计划变化和院校专业信息，提供院校与专业两个方向的录取概率参考、风险标签、历史依据和 AI 解释。

产品表达统一遵循：

```text
数据是依据，AI 是辅助；预测是参考，决策在人。
```

系统不替代教育考试院、高校招生办、官方志愿填报系统或考生本人做出最终决策，不提供保录、代填、内部名额或录取承诺。

## 2. 系统 1.0 目标

系统 1.0 采用手机端优先、Web/H5 兼容的可点击交互原型，完整呈现一条清晰可用的核心链路：

```text
输入考生信息
→ 校验省份规则
→ 换算全省位次
→ 查看院校冲稳保推荐
→ 查看院校近三年数据
→ 查看专业组与专业预测
→ 生成 AI 辅助分析
→ 加入志愿单
→ 查看梯度诊断
```

首版聚焦广东省普通类物理方向的完整演示。其他省份在首页保留入口，并标注“数据接入中”，不生成虚假的完整预测结果。

## 3. 用户价值

### 3.1 考生

- 快速判断当前分数和位次可以重点关注哪些院校。
- 区分院校整体录取机会与具体专业录取风险。
- 避免只看最低分，不看位次、专业组和选科要求。
- 将目标院校加入志愿单，检查冲刺、稳妥和保底梯度。

### 3.2 家长

- 看懂每个概率背后的历史数据和风险原因。
- 通过近三年位次、招生计划和专业组要求判断结果是否可信。
- 获得适合家庭讨论的结构化参考信息。
- 明确区分“辅助分析”与“录取承诺”。

## 4. 竞品启发与产品收束

成熟志愿填报产品通常同时覆盖院校库、专业库、智能推荐、职业测评、升学资讯、直播咨询和志愿方案管理。优志愿公开产品说明中已包含录取分数线、位次、当年招生计划、目标专业、冲稳保推荐、院校专业组模式和录取概率预测等能力。

系统 1.0 不建设复杂门户，不引入职业测评、咨询老师、资讯流、直播课程、复杂会员和一键代填能力。首页只解决三个问题：

```text
输入什么信息？
可以得到什么结果？
结果为什么只能作为参考？
```

结果页不只展示单点概率，而是统一展示：

```text
概率参考区间
+ 风险标签
+ 数据置信度
+ 近三年趋势
+ 招生计划变化
+ AI 解释
+ 官方资料核对提醒
```

## 5. 广东省演示规则

广东省普通类志愿采用“院校专业组 + 专业（类）”模式。一个院校专业组是一个独立志愿，也是一个投档单位。同一专业组内的专业具有一致的选考科目要求。

系统 1.0 对广东省普通类物理方向执行以下前置校验：

1. 首选科目必须匹配专业组要求。
2. 再选科目必须满足“不限”“必须包含一门”“必须同时包含两门”或“满足任意一门”等规则。
3. 选科不符合要求时，直接显示“不可报考”，不进入概率计算。
4. 数据少于两年、招生计划变化过大或专业样本过小时，显示“数据不足”。
5. 页面提醒用户核对高校招生章程、身体条件限制、专业录取规则、是否服从调剂和官方招生计划。

## 9. 预测引擎设计

### 9.1 分层原则

DeepSeek 不直接决定录取概率。预测引擎分为三层：

```text
规则层
→ 校验省份模式、批次、院校专业组和选科要求

基础算法层
→ 根据历史数据计算参考概率、概率区间、风险标签和置信度

AI 解释层
→ 根据结构化结果生成原因说明、风险提示和核对清单
```

### 9.2 基础算法输入

```json
{
  "year": 2026,
  "province": "广东",
  "subject_type": "普通类物理",
  "selected_subjects": ["化学", "生物"],
  "score": 612,
  "rank": 14800,
  "target_university": "华南理工大学",
  "target_group": "202专业组",
  "target_major": "软件工程",
  "batch": "本科批",
  "history": [
    {
      "year": 2023,
      "min_score": 607,
      "min_rank": 15400,
      "avg_rank": 13200,
      "plan_count": 22
    },
    {
      "year": 2024,
      "min_score": 610,
      "min_rank": 14900,
      "avg_rank": 12800,
      "plan_count": 22
    },
    {
      "year": 2025,
      "min_score": 611,
      "min_rank": 14600,
      "avg_rank": 12600,
      "plan_count": 24
    }
  ],
  "plan_change_rate": 0.09,
  "data_completeness": 0.86
}
```

### 9.3 风险标签

| 参考概率 | 标签 |
| --- | --- |
| 90% - 100% | 保底 |
| 75% - 89% | 稳妥 |
| 60% - 74% | 稳妥偏冲 |
| 40% - 59% | 冲刺 |
| 20% - 39% | 高风险冲刺 |
| 0% - 19% | 不建议作为主要志愿 |
| 数据不足 | 历史数据少于 2 年或规则变化明显 |

阈值由后端配置管理，前端不写死业务判断。

### 9.4 置信度

置信度表示预测依据是否充分，不等于录取概率。

| 置信度 | 判断条件 |
| --- | --- |
| 高 | 至少 4 年完整数据，计划稳定，批次规则稳定 |
| 中 | 约 3 年数据，少量字段缺失，计划轻微变化 |
| 低 | 仅 1-2 年数据，新专业，小样本，计划或规则变化明显 |

## 10. DeepSeek 接入设计

### 10.1 输入边界

DeepSeek 只接收经过规则校验和基础算法计算后的结构化数据。输入中必须包含概率区间、风险标签、置信度、近三年历史、招生计划变化、选科校验结果和数据来源。

DeepSeek 不允许：

- 编造院校、专业、分数线、位次或招生计划。
- 修改基础算法生成的概率。
- 承诺录取结果。
- 省略数据不足提示。
- 把预测结果描述为官方结论。

### 10.2 JSON 输出

请求使用 JSON 输出模式：

```json
{
  "response_format": {
    "type": "json_object"
  }
}
```

输出结构：

```json
{
  "summary": "当前位次接近该专业组近三年最低录取位次，具备一定机会，但热门专业仍需谨慎。",
  "reasoning": [
    "近三年最低录取位次整体波动较小，历史参考价值较高。",
    "当前位次略低于近三年平均录取位次，但接近最低录取区间。",
    "今年计划人数较上一年度小幅增加，风险略有下降。"
  ],
  "risk_warning": [
    "热门专业竞争强度高于院校整体。",
    "正式填报前需核对招生章程、专业组选科要求和是否服从调剂。"
  ],
  "next_actions": [
    "同时配置 2-3 所参考概率更高的保底院校。",
    "查看专业组内不同专业的概率区间。"
  ],
  "disclaimer": "本结果基于历年公开数据和 AI 分析生成，仅供志愿填报辅助参考，不代表最终录取结果。"
}
```

### 10.3 兜底策略

DeepSeek 官方文档说明 JSON 输出模式可能偶发返回空内容，因此系统必须提供固定解释模板。

| 异常 | 页面行为 | 额度处理 |
| --- | --- | --- |
| AI 响应超时 | 展示基础概率和固定解释模板 | 不扣额度或自动退回 |
| AI 返回空内容 | 展示基础概率和固定解释模板 | 不扣额度或自动退回 |
| JSON 解析失败 | 展示基础概率和固定解释模板 | 不扣额度或自动退回 |
| 字段缺失或越界 | 丢弃异常字段，使用固定解释模板 | 不扣额度或自动退回 |
| 网络中断 | 保留已输入条件，支持重新生成 | 不扣额度 |

## 12. 交互状态

| 场景 | 页面反馈 |
| --- | --- |
| 未填写必填项 | 在对应字段下方提示缺失内容 |
| 分数超出省份总分范围 | 提示输入有效分数 |
| 已填写位次 | 优先使用用户确认位次 |
| 一分一段表缺失 | 标记“当前按历史数据估算，准确性较低” |
| 选科不匹配 | 显示“不可报考”，禁止加入志愿单 |
| 历史数据不足 | 显示“数据不足”，不强行输出确定性结论 |
| AI 分析生成中 | 先展示基础结果，AI 卡片显示生成状态 |
| 重复查看同一结果 | 使用缓存，不重复扣除额度 |
| 加入志愿单成功 | 底部轻提示，按钮变为“已加入” |
| 志愿单保底不足 | 显示橙色风险提示和调整建议 |

## 13. AI 使用额度与费用说明

系统 1.0 只保留一个收费概念：

```text
AI 使用额度
```

消耗额度：

- 生成新的院校 AI 分析。
- 生成新的专业 AI 分析。
- 生成志愿单梯度诊断。

不消耗额度：

- 查看院校基础信息。
- 查看已经生成的结果。
- 查看近三年历史数据。
- 查看数据来源。
- 管理志愿单。
- 查看免责声明和协议。

确认弹窗固定说明：

```text
本次将消耗 1 次 AI 使用额度。
AI 分析基于历年公开数据生成，仅供辅助参考，不代表最终录取结果。
```

AI 使用费用说明页固定展示：

```text
AI 使用费用用于支持院校预测解释、专业预测解释和志愿单梯度诊断。
购买的是 AI 辅助分析服务，不是录取保证，不提供保录、代填志愿或官方录取承诺。
```

费用说明页提供“免费体验”“轻量 AI 使用包”“标准 AI 使用包”三类展示项。原型仅模拟确认支付和额度到账，不接入真实支付渠道。

支付确认区域必须提供：

- 当前购买的 AI 使用次数。
- 金额和有效期。
- 扣费范围。
- 未生成有效结果时的自动退回规则。
- 《用户协议》《隐私政策》《AI 使用说明》《预测结果免责声明》入口。
- “确认支付表示已阅读并同意相关协议”的短版提示。

## 14. 数据采集与检索

### 14.1 数据采集原则

院校、专业、录取数据和招生计划必须来自可追溯的数据源。系统优先使用：

1. 省教育考试院公开数据。
2. 高校招生网、招生章程和官方招生计划。
3. 教育部阳光高考平台等权威公开入口。
4. 已经人工核验的数据导入表。

### 14.2 检索流程

系统不允许让 DeepSeek 在用户发起预测时临时搜索网页并直接生成事实。正式系统采用：

```text
定时采集公开数据
→ 保存原始来源链接与采集时间
→ 结构化清洗
→ 自动校验
→ 管理后台抽样复核
→ 发布数据版本
→ 用户查询时读取已发布版本
```

DeepSeek 可以根据已发布的结构化数据生成解释，但不得补充数据库中不存在的学校信息、分数线、位次和招生计划。

### 14.3 数据版本

每次预测记录至少保留：

```text
数据版本
算法版本
Prompt 版本
来源链接
来源更新时间
生成时间
```

招生计划、专业组规则或数据源更新后，对应缓存自动失效。

## 17. 验收标准

### 17.1 功能验收

- 首页可输入广东省、普通类物理、再选科目、分数或位次。
- 分数输入后可进入位次确认页。
- 位次确认后可查看至少 6 所院校的冲稳保推荐。
- 推荐列表可按全部、冲刺、稳妥、保底筛选。
- 院校详情可查看概率区间、置信度、近三年趋势、招生计划、来源和免责声明。
- 专业预测可展示专业组、选科要求、计划人数、专业概率区间和风险标签。
- 选科不匹配的专业不可加入志愿单。
- AI 分析模拟异步生成，并展示固定结构化内容。
- 志愿单可添加、移除院校专业组，并展示梯度诊断。
- AI 使用费用说明页可展示次数包、服务边界、协议入口和模拟确认支付。
- 模拟确认支付后可展示额度到账状态。
- 非广东省入口显示“数据接入中”，不伪造完整结果。

### 17.2 体验验收

- 360px 至 430px 宽度下无横向滚动。
- 手机端关键操作可在底部固定操作区完成。
- 结果页首屏可看到院校名称、概率区间、风险标签和置信度。
- 所有核心页面均展示“仅供辅助参考”提示。
- Web/H5 端页面居中展示，宽屏下不出现过度拉伸。
- 所有按钮具备可见的点击反馈。

### 17.3 合规验收

- 页面中不出现“必录”“保录”“百分百准确”“官方预测”“内部数据”等表述。
- AI 解释不修改基础算法生成的概率。
- AI 分析失败时仍可查看基础结果。
- 分享或演示内容不默认展示真实考生敏感信息。
- 页面提醒用户核对教育考试院、高校招生章程和官方招生计划。

## 18. 原型范围边界

系统 1.0 原型不包含：

- 真实 DeepSeek 密钥和真实 AI 计费。
- 真实支付渠道和支付回调。
- 运行时自由网页搜索。
- 职业兴趣测评。
- 咨询老师与直播课程。
- 开放式 AI 聊天。
- 自动提交志愿。
- 复杂会员体系。
- 全量省份和全量院校数据。

## 19. 数据建设边界

原型完成后，真实系统上线前至少需要接入：

1. 各省教育考试院公开的一分一段表。
2. 各省批次线和报考模式。
3. 高校代码、院校专业组代码和专业代码。
4. 近三至五年院校与专业最低分、最低位次、平均分和平均位次。
5. 当年招生计划、选科要求、学费、学制和专业限制。
6. 数据来源链接、更新时间和校验状态。
7. 算法版本、Prompt 版本、预测记录和用户确认记录。
